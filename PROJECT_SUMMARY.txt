================================================================================
API ENDPOINTS
================================================================================

PUBLIC ROUTES (No Authentication)
--------------------------------------------------------------------------------
GET     /api/health                      Health check
POST    /api/v1/register                 Register new user (requires username)
POST    /api/v1/login                    Login
GET     /api/v1/roles                    Get available roles

PASSWORD RESET
--------------------------------------------------------------------------------
POST    /api/v1/password/forgot          Request reset code
POST    /api/v1/password/verify-code     Verify code
POST    /api/v1/password/reset           Reset password

AUTHENTICATED ROUTES
--------------------------------------------------------------------------------
GET     /api/v1/user                     Get current user
POST    /api/v1/logout                   Logout
POST    /api/v1/logout-all               Logout from all devices
GET     /api/v1/onboarding/status       Onboarding progress
POST    /api/v1/onboarding/name         Save name
POST    /api/v1/onboarding/birthday     Save birthday
POST    /api/v1/onboarding/represent    Save recruiter representation

EMAIL VERIFICATION
--------------------------------------------------------------------------------
POST    /api/v1/email/send-verification  Send verification code
POST    /api/v1/email/verify             Verify email
GET     /api/v1/email/status             Get verification status

POSTS API (Requires verified email)
--------------------------------------------------------------------------------
GET     /api/v1/posts                    Get feed (global/following)
POST    /api/v1/posts                    Create post
GET     /api/v1/posts/{id}               Get single post
PUT     /api/v1/posts/{id}               Update post (owner only)
DELETE  /api/v1/posts/{id}               Delete post (owner only)

ROLE-BASED ROUTES (Requires verified email)
--------------------------------------------------------------------------------
GET     /api/v1/gamer                    Gamer dashboard (role: gamer)
GET     /api/v1/recruiter                Recruiter dashboard (role: recruiter)

================================================================================
REQUEST/RESPONSE EXAMPLES
================================================================================

REGISTER
--------------------------------------------------------------------------------
POST /api/v1/register
{
    "name": "Player Name",
    "username": "player_name",
    "email": "player@example.com",
    "password": "password123",
    "password_confirmation": "password123",
    "role": "gamer"
}

Notes:
- username: required, 3-20 characters, unique, alphanumeric and underscore only
- Username format: /^[a-zA-Z0-9_]+$/

Response (201):
{
    "success": true,
    "message": "User registered. Please verify your email.",
    "data": {
        "user": {
            "id": 1,
            "name": "Player Name",
            "username": "player_name",
            "email": "player@example.com",
            "role": "gamer"
        },
        "token": "1|abc123...",
        "token_type": "Bearer",
        "verification_required": true,
        "code": "123456"
    }
}

LOGIN
--------------------------------------------------------------------------------
POST /api/v1/login
{
    "email": "player@example.com",
    "password": "password123"
}

Response (200):
{
    "success": true,
    "message": "Login successful",
    "data": {
        "user": { ... },
        "token": "1|abc123...",
        "token_type": "Bearer"
    }
}

PASSWORD FORGOT
--------------------------------------------------------------------------------
POST /api/v1/password/forgot
{
    "email": "player@example.com"
}

POST /api/v1/password/verify-code
{
    "email": "player@example.com",
    "code": "123456"
}

POST /api/v1/password/reset
{

    "reset_token": "abc123...",
